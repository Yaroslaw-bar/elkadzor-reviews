# Elkadzor Review Monitor

**Мониторинг и аналитика отзывов о компании «Элькадор» с нескольких площадок в одном интерфейсе.**

Проект собирает отзывы с сайта компании, Яндекс.Карт и других источников, сохраняет их в локальную базу, уведомляет о новых сообщениях в Telegram и показывает статистику через веб-дашборд.
![Elkadzor Review Monitor](images/Снимок%20экрана%202026-07-03%20015405.png)
---

## Название

**Elkadzor Review Monitor** — инструмент репутационного мониторинга для компании «Элькадор».

---

## Для кого

Для владельцев и менеджеров локального бизнеса, которым важно:

- не пропускать новые отзывы;
- быстро реагировать на негатив и благодарить за позитив;
- видеть общую репутацию компании в разрезе площадок;
- понимать, какие источники работают стабильно, а какие требуют доработки.

Проект сделан как практический инструмент: без сложных сервисов и подписок, работает локально на Python.

---

## Какие задачи решает

1. **Сбор отзывов в одно место**
   Отзывы разбросаны по сайту компании, Яндекс.Картам, 2GIS, Plaso, RemKarta и другим площадкам. Скрипт собирает их централизованно в SQLite-базу.

2. **Уведомления о новых отзывах**
   При появлении нового отзыва бот отправляет сообщение в Telegram с автором, рейтингом, текстом и ссылкой.

3. **Аналитика и дашборд**
   Streamlit-дашборд показывает:
   - общее количество отзывов и средний рейтинг;
   - распределение по площадкам;
   - фильтры по источнику, рейтингу и периоду;
   - экспорт в CSV.
   - ![CSV](images/Снимок%20экрана%202026-07-03%20020457.png)
   - динамику по датам;
   - ![динамику по датам](images/Снимок%20экрана%202026-07-03%20015414.png)
   - распределение по площадкам;
   - ![распределение по площадкам](images/Снимок%20экрана%202026-07-03%20025116.png)


4. **Автоматизация**
   Планировщик запускает сбор раз в сутки (или с любым интервалом), чтобы мониторинг работал без ручного участия.

5. **Прозрачность источников**
   В конфиге и таблице видно, какие площадки работают стабильно, а какие блокируются капчей или требуют доработки.

## Стек

- Python 3
- `requests` + `beautifulsoup4` + `lxml` — статический парсинг
- `playwright` — динамические источники
- `sqlite3` — хранилище
- `python-telegram-bot` — уведомления
- `apscheduler` — планировщик
- `streamlit` + `pandas` — дашборд
- `python-dotenv` — переменные окружения

## Установка

PowerShell:

```powershell
cd C:\project\elkadzor-reviews\project
python -m venv .venv
.\.venv\Scripts\pip install -r requirements.txt
```

Скопируй `.env.example` в `.env` и добавь токен бота и `chat_id`:

```powershell
cd C:\project\elkadzor-reviews\project

```

Пример `.env`:

```env
TELEGRAM_BOT_TOKEN=your_bot_token_here
TELEGRAM_CHAT_ID=your_chat_id_here
```

## Использование

### Разовый сбор отзывов

```powershell
& C:\project\elkadzor-reviews\project\.venv\Scripts\python.exe C:\project\elkadzor-reviews\project\collect_reviews.py
```

Скрипт проверит сайт Элькадор и Яндекс.Карты, добавит новые отзывы в `reviews.db` и при необходимости отправит уведомление в Telegram.

### Постоянный мониторинг

Запускать сбор раз в сутки:

```powershell
cd C:\project\elkadzor-reviews\project
.\.venv\Scripts\python -c "from scheduler import start_scheduler; start_scheduler(24)"
```

### Дашборд

```powershell
cd C:\project\elkadzor-reviews\project
$env:STREAMLIT_SERVER_HEADLESS="true"
.\.venv\Scripts\python -m streamlit run dashboard.py
```

Откроется на `http://localhost:8501`.

## Структура проекта

| Файл | Назначение |
|---|---|
| `project/config.py` | Источники отзывов, заголовки, переменные окружения |
| `project/parser.py` | Сбор статических страниц (сайт Элькадор) |

| `project/notifier.py` | Отправка уведомлений в Telegram |
| `project/scheduler.py` | Планировщик и одиночный запуск |
| `project/dashboard.py` | Веб-интерфейс Streamlit |
| `project/collect_reviews.py` | Точка входа для ручного сбора |
| `project/requirements.txt` | Зависимости |

## Как добавить новую площадку

1. Добавь источник в `config.py`:

```python
"new_source": {
    "name": "Название площадки",
    "url": "https://example.com/reviews",
    "type": "static",  # или "dynamic"
}
```

2. Реализуй парсер:
   - статический — в `parser.py`;
   - динамический — в `dynamic_parser.py`.

3. Подключи парсер в `collect_reviews.py` через словарь `parsers`.

4. Добавь источник в `ACTIVE_SOURCES`, если он готов к регулярному сбору.
